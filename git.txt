1. git은 버전관리 시스템
   commit을 할 떄 마다 버전이 1씩 올라간다
   그런데 작업중 실수할 떄 버전이 올라가면 버전관리가 어렵다
   - login담당자가 테스트를 위해 login.html까지 만든다음 실수로 html까지 commit
     version 1. 로그인 구현
     version 2. login.html 삭제
  위와 같은 상황을 줄이기위해 staging 영역이라는 임시작업영역을 도입

2. git init
  - 현재 폴더를 git 프로그램으로 관리하겠다

3. git add와 git reset으로 스테이징 영역에 추가/삭제한다
  - 실수를 해도 버전으로 남지 않는다

4. 작업이 일단락되면 commit한다
  - git commit -m "작업내용" : 반드시 메시지를 남긴다
